# Privacy Policy  
_Last updated: 2025_

This Privacy Policy explains how **PDF Converter Bot Team** ("we", "us", "our") collects, processes, stores, and protects information when you use the **PDF Converter Bot** ("the Bot") on Telegram.

By using the Bot, you agree to the practices described in this Policy.

---

## 1. Information We Collect

### 1.1. Telegram User Data
We automatically receive basic Telegram user information provided by Telegram:
- User ID  
- Username (if available)  
- Language code  
- Chat ID  

This information is required for the Bot to function.

### 1.2. Uploaded Files
When you send files (documents, images, PDFs, scans), we process them solely to provide the requested features, including:
- Conversion to PDF  
- Compression  
- OCR (text extraction)  
- Page editing  
- Watermarking  

We do **not** manually review, analyze, or share the content of these files.

### 1.3. Payment Information
If you subscribe to PRO features, we receive:
- Payment status (success/failed)  
- Transaction identifier  
- Subscription status and expiration date  

We **never** receive or store credit card details.  
All payments are processed securely via **Stripe** or Telegram Payments.

### 1.4. Technical Logs
To maintain service quality, we may temporarily store technical logs:
- File size  
- Processing times  
- Error messages  

Logs do **not** contain the content of your files.

## 3. File Storage and Deletion

### 3.1. Temporary Processing
Uploaded files are stored **temporarily** on secure servers for processing.

Typical retention time: **from a few seconds to under 1 hour**.

### 3.2. Automatic Deletion
Files are deleted automatically after processing is completed or if an error occurs.

### 3.3. No Long-Term Storage
We do NOT:
- Store files permanently  
- Use user files for training  
- Create backups of user files

## 4. Data Sharing
We may share limited technical data only with:
- **Telegram**, as part of the Telegram API  
- **Stripe**, for payment processing  
- **Hosting providers**, strictly for temporary file processing  

No user content is shared beyond operational requirements.
